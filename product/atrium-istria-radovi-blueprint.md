# Atrium Istria Radovi Blueprint

Дата: 2026-03-25  
Статус: structural blueprint

## 1. Page goal

`Radovi` должна подтверждать реальность и уровень компании через case-based presentation, а не через хаотичную фотогалерею.

## 2. Page structure

### Block 1. Intro

- short page intro
- what kind of work is shown here
- trust framing

### Block 2. Case grid

Every case card should include:

- object type
- location if useful
- what was
- what we did
- result
- key image

### Block 3. Expanded case logic

If page design allows:

- open more details for a case
- show before/process/after sequence
- list related services

### Block 4. Final CTA

- link back into conversation
- phone / contact

## 3. Card model

Minimum fields:

- title
- object type
- location
- before summary
- work summary
- result summary
- key visual
- linked service

## 4. Best current source folders

- `домик`
- `Мансарда до:после`
- `Санузлы`
- selected supporting folders for smaller cards

## 5. Selection rules

Use cases where:

- there is visible transformation
- there is understandable story
- there is enough image quality
- result looks controlled and convincing

Avoid cases where:

- visual story is incomplete
- photos are too chaotic
- there is no clear before/result logic
