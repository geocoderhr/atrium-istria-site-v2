# Atrium Istria Visual Direction Brief

Дата: 2026-03-25  
Статус: working design brief  
Основа: PRD v1, research, current photo library, selected references

## 1. Core direction

Atrium Istria должен выглядеть как современная, собранная и технологичная строительная компания, которая уверенно ведет объект и коммуникацию.

Необходимый визуальный гибрид:

- architectural
- layered
- glass-panel
- premium-but-practical
- interface-driven
- conversation-ready

Сайт не должен выглядеть:

- как обычный строительный лендинг;
- как SaaS-dashboard;
- как luxury-studio portfolio без прикладной пользы;
- как chatbot product.

## 2. Main references and their roles

### Primary references

- `AXIOMA`
  - роль: архитектурная собранность, взрослая строгость, clean hierarchy
- `Travel Website Landing Page`
  - роль: glass panels, прозрачные окна, мягкие светлые поверхности
- `Landbot`
  - роль: conversational hero logic
- `Solra / Real Estate`
  - роль: premium landing composition, polished visual rhythm

### Supporting references

- `Hotel Booking Website`
  - роль: polished atmosphere, calm premium mood
- `Real Estate Landing Page`
  - роль: imagery framing, first impression, section pacing

## 3. Color direction

### Base palette logic

- Primary base: deep anthracite
- Secondary surfaces: soft light neutrals
- Accent: warm orange
- Optional support tones: muted stone, warm grey, soft off-white

### Color behavior

- Anthracite задает уверенность и глубину, но не должен уходить в чистый черный.
- Теплый оранжевый нужен не как агрессивный CTA-color, а как цвет энергии, материала и живого характера.
- Светлые стеклянные панели должны смягчать интерфейс и давать воздух.
- Визуальный тон должен быть глубоким и спокойным, а не мрачным.

### Palette constraints

- no pure-black dominance
- no purple bias
- no cold-tech blue as main accent
- no overly glossy luxury-gold behavior

## 4. Typography direction

### General principle

Типографика должна быть:

- clean
- confident
- adult
- practical
- premium without exhibitionism

### Required behavior

- Крупный и уверенный hero headline
- Четкая иерархия между headline, support copy, labels and UI text
- Спокойные плотные подписи в карточках и системных панелях
- Хорошая читаемость на mobile-first layout

### Typographic tone

- not editorial-artsy
- not startup-generic
- not classic contractor-heavy

## 5. Hero layers

Hero должен быть не просто большим баннером, а многослойной системой.

### Layer 1: Background

- приглушенное фото реального объекта или архитектурного пространства
- фото должно работать как атмосфера, а не как крикливая обложка
- допустим мягкий color grading и затемнение

### Layer 2: Anthracite atmosphere

- темная подложка
- глубина
- controlled contrast

### Layer 3: Glass panel

- светлая полупрозрачная панель
- мягкий blur
- clean frame language
- без тяжелых borders и без glossy gimmicks

### Layer 4: Core content

- H1
- subhead
- первая человеческая фраза
- input panel

### Layer 5: Supporting structure

- secondary containers
- trust accents
- возможно короткие structured facts or proof cues
- все это должно усиливать ощущение системы, а не загромождать экран

## 6. Conversational panel rules

Панель разговора на hero — один из ключевых элементов сайта.

### It must be

- light
- adult
- calm
- premium
- integrated into hero composition

### It must NOT be

- chatbot widget
- black messenger bubble
- SaaS product card
- button-heavy flow builder

### Tone of interaction

- human
- concise
- practical
- inviting but not salesy

### UX principle

Пользователь должен почувствовать:

- “я могу нормально описать свой объект”
- “мне ответят по делу”
- “здесь есть порядок”

## 7. Container language

Сайт должен использовать panels/containers как часть бренда.

### Container characteristics

- ровные геометрии
- clean corners
- subtle separation from background
- soft depth
- clear internal spacing

### What they should communicate

- system
- process
- structure
- control

### What to avoid

- clutter
- floating random cards
- excessive outlines
- gamified UI feeling

## 8. Header direction

Header должен выглядеть как строгая control bar.

### Must contain

- logo
- language switch
- phone
- `Usluge / Radovi / Kontakt`

### May contain

- email on desktop

### Must feel like

- calm
- precise
- high-trust
- non-decorative

## 9. Image treatment

### General rules

- Фото должны поддерживать ощущение реального бизнеса и доверия.
- Изображения не должны превращать сайт в хаотичную галерею.
- Hero images и project images требуют разной обработки.

### Hero image treatment

- darker
- calmer
- more atmospheric
- integrated with anthracite overlay and glass panel

### Project image treatment

- clearer
- more documentary
- more proof-oriented

### Service image treatment

- scenario-supportive
- practical
- readable even on mobile

## 10. Direction summary

Формула дизайна Atrium Istria:

`architectural discipline + layered interface + soft glass panels + real construction proof + human conversation entry`

Это должно давать ощущение:

- organized company
- practical intelligence
- real process control
- premium trust
- calm confidence
